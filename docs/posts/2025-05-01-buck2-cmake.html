Why do this?

My goal that led to this was that I wanted to understand how to switch a C++ project from cmake to buck2. Doing this incrementally, I can handle most of the vendored dependencies, but a one in particular seemed to be tightly integrated into cmake and used a lot of features from cmake to generated the final header and library. Instead of figuring out this whole conversion and potentially jumping down a long rabbit hole, I just wanted buck2 to run cmake for that dependency, and hook the output into the rest.

My background

Example: libdivsufsort

Setup: C++ Project with libdivsufsort as a dependency

Switching to buck2

Initialize repo

buck2 init --git

genrule for cmake

genrule(
    name = "libdivsufsort/cmake",
    srcs = ["."],
    cmd = "echo TODO >$OUT"
    out = "TODO.txt"
)

Writing the cmd

genrule(
    name = "libdivsufsort/cmake",
-   srcs = ["."],
+   srcs = ["libdivsufsort"],
-   cmd = "echo TODO >$OUT"
+   cmd = """
+OUT=$PWD/$OUT
+cp -rL $SRCS $TMP
+(
+   $TMP/libdivsufsort
+   mkdir build
+   cd build
+   cmake ..
+   make
+   make install
+   
+)
-   out = "TODO.txt"
+   outs = {
+       "header": ["include/divsufsort.h"],
+       "lib": ["lib/libdivsufsort.a"],
+       "lib64": ["lib/libdivsufsort64.a"],
+   }
)

Linking to the binary

# BUCK
cxx_library(
    name = "libdivsufsort",
    exported_headers = [":libdivsufsort/cmake[header]"],
    public_include_directories = [":libdivsufsort/cmake[header]"],
    exported_linker_flags = [
        "$(location :libdivsufsort/cmake[lib])",
        "$(location :libdivsufsort/cmake[lib64])",
    ]
)

Testing it all works

// test.cpp
#include <iostream>
#include "libdivsufsort.h"

int main(void) {
    // TODO copy from example
}
# BUCK
cxx_binary(
    name = "test",
    srcs = ["test.cpp"],
    deps = [":libdivsufsort"]
)

Troubleshooting

Notes about things that went wrong and how I fixed them

  • buck2 is too good at caching things, and changes in the the BUCK are sometimes not reflected in buck2 build, and usually if I’m iterating on the BUCK file I need to force it to look at the updated BUCK file. To do this, I usually need to run buck2 killall to force it to re-read the BUCK file.
  • Target patterns (e.g. “:libdivsufsort”) and string parameter macros (e.g. “$(location :libdivsufsort[header]”)) can’t be used in all parameters for rules. If the compiler is complaining about finding files, undefined references, etc. Look at the argfiles that buck2 generates and see if it is getting expanded correctly.
